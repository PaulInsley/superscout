# REPLIT AGENT PROMPT — SuperScout: Ownership Spectrum Visual on Captain Cards

## CONTEXT

SuperScout’s Captain Picker cards currently show an AI-generated ownership_context coaching line below the ownership percentage. The coaching text is good but lacks visual punch. We’re adding a small visual element — a spectrum bar — that shows at a glance whether this captain pick goes “with the crowd” or “against the crowd.”

This replaces nothing — the coaching text stays. The spectrum sits ABOVE the coaching text and gives it visual weight.

## TASK

### 1. Build an OwnershipSpectrum component

A compact visual element with three parts:

**Part A — Labels row:** Two small labels at either end of the spectrum:

- Left: “With the crowd” (use the app’s indigo/primary colour, muted)
- Right: “Against the crowd” (use text-muted colour)
- Font: 10px, uppercase, 600 weight, letter-spacing 0.5px

**Part B — The track:** A horizontal bar, 6px tall, with a subtle gradient from indigo-light (left) to grey (right). Border-radius 3px. Full width of the card minus padding.

**Part C — The dot:** A 16px circle positioned on the track based on ownership percentage. The dot has:

- Background: indigo/primary colour
- 3px white border
- Small box shadow for depth
- Position: calculated from ownership percentage, but INVERTED — low ownership = far right (against the crowd), high ownership = far left (with the crowd)

**Positioning formula:**

- `left: (100 - ownership_percent)%`
- So 45% owned = dot at 55% from left (slightly right of centre)
- 2.9% owned = dot at 97% from left (far right, against the crowd)
- 70% owned = dot at 30% from left (well into “with the crowd”)

**Part D — Coaching text:** The existing AI-generated ownership_context text sits directly below the track. Italic, 12.5px, text-secondary colour. This is already being generated by Claude — just move it inside this component.

### 2. Container styling

Wrap all four parts in a container with:

- Background: grey-50 / surface-alt (the lightest card background)
- Border-radius: 10px
- Padding: 10px 14px
- Margin-bottom: 14px

This gives the spectrum its own visual section on the card, distinct from the fixture ticker above and the upside/risk below.

### 3. Position on the captain card

The spectrum component sits:

- BELOW the confidence badge + ownership percentage row
- ABOVE the Upside/Risk section
- The fixture ticker stays where it is (above the confidence row)

Card layout order should be:

1. Player name + match + EXP PTS
1. Fixture ticker (NEXT 5)
1. Confidence badge + ownership %
1. **Ownership Spectrum (NEW)** — track, dot, coaching text
1. Upside / Risk
1. Lineup Changes (if applicable)
1. Vibe commentary quote

### 4. Remove the old standalone coaching text

The ownership_context coaching line was previously displayed as a standalone italic line below the ownership percentage. Remove that standalone line — the same text now lives inside the spectrum component. Don’t remove the ownership_context field from the response or the parsing — just move where it displays.

### 5. Handle edge cases

- If ownership_context is null (ownership data missing): hide the entire spectrum component. No track, no dot, no text.
- If ownership percentage is 0 or not available: hide the entire spectrum component.
- If ownership is above 95%: cap the dot position so it doesn’t sit on top of the left label. Minimum left position: 5%.
- If ownership is below 1%: cap the dot position so it doesn’t sit on top of the right label. Maximum left position: 95%.

### 6. Do NOT change anything else

- Do NOT modify the fixture ticker
- Do NOT modify the confidence badges, upside/risk text, or vibe commentary
- Do NOT modify the Transfer Advisor cards — this is Captain Picker only
- Do NOT modify the AI prompt or response schema — the ownership_context field is already being generated correctly
- Do NOT modify the pre-generation pipeline or staleness check
