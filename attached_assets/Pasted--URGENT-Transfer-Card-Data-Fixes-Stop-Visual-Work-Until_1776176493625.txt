## Problem 2: The impact number is nonsensical

**What I see:**

- Guéhi card shows “+42 pts vs Saliba” over 3 gameweeks
- Casemiro card shows “-2 pts vs Barkley” but the coaching text is actively recommending the move

**Why this is wrong:**

- +42 points differential over 3 gameweeks is absurd. That would mean Guéhi scores ~50 points and Saliba scores ~8 points in 3 GWs. No player averages 17 points per gameweek.
- A transfer card recommending a move that loses you 2 points makes no sense. If the impact is negative, either the recommendation shouldn’t exist or the number is wrong.

**What to do:**

Step 1: Find the code that calculates the impact number shown on the transfer card. Search for where “+42” or the impact score is generated. Tell me EXACTLY how it’s currently calculated — show me the code or logic.

Step 2: Is the AI generating this number? If so, that’s the root cause. The AI will hallucinate numbers. The impact score must be calculated deterministically from FPL data, not generated by the AI.

Step 3: Implement this deterministic calculation:

```
expected_points(player, N_gameweeks) = sum over next N gameweeks of:
  form × fixture_modifier × minutes_probability
```

Where:

- `form` = the `form` field from FPL API (the CORRECT one from Problem 1 — the string, parsed as a float)
- `fixture_modifier` depends on FDR for that fixture:
  - FDR 1 = multiply by 1.20
  - FDR 2 = multiply by 1.10
  - FDR 3 = multiply by 1.00
  - FDR 4 = multiply by 0.85
  - FDR 5 = multiply by 0.70
- `minutes_probability` = from the FPL API `chance_of_playing_next_round` field:
  - If null or missing: use 1.0 (assume they play)
  - If 0: use 0.0 (injured/unavailable)
  - If 25: use 0.25
  - If 50: use 0.50
  - If 75: use 0.75
  - If 100: use 1.0
- For DGW gameweeks: the player gets TWO fixture entries, each with its own FDR modifier. Both count.
- For BLANK gameweeks: the player gets 0 points for that gameweek.

The IMPACT NUMBER displayed on the card is:

```
For free transfers:
  impact = expected_points(IN_player, 3 gameweeks) - expected_points(OUT_player, 3 gameweeks)

For hit transfers (-4 points):
  impact = expected_points(IN_player, 5 gameweeks) - expected_points(OUT_player, 5 gameweeks) - 4
```

Step 4: Display the timeframe:

- Free transfer: “Over next 3 gameweeks”
- Hit transfer: “Over 5 GWs · evens in X” where X = the breakeven gameweek count

Breakeven = smallest N where expected_points(IN, N) - expected_points(OUT, N) >= hit cost. If breakeven > 8, flag it in the coaching text.

Step 5: Sanity check. Run the calculation for the Guéhi vs Saliba transfer. Show me:
