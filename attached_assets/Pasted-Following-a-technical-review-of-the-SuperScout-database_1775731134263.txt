Following a technical review of the SuperScout database schema, I need to add three things to Supabase. Go to the SQL Editor and run the following changes.
Addition 1: Create a new table called recommendation_options. This stores each individual option that was shown within a recommendation as a separate row, rather than inside a JSON blob.
Fields: id (UUID, primary key), recommendation_id (UUID, foreign key to recommendations), option_rank (integer — 1 to 5, the position this option appeared), player_id (integer, nullable), option_type (text), expected_points (decimal, nullable), confidence_score (decimal, nullable — 0 to 1), confidence_label (text — HIGH, MEDIUM, or SPECULATIVE), upside_text (text, nullable), risk_text (text, nullable), is_superscout_pick (boolean, default false), created_at (timestamptz, default now()).
Then update the user_decisions table: add a column called recommendation_option_id (UUID, nullable, foreign key to recommendation_options.id). This lets decisions reference a specific option directly rather than a position in a JSON blob.
Addition 2: Create a new table called inference_context. This records which version of the AI engine, persona prompt, and model was used for each recommendation — essential for proving improvement over time.
Fields: id (UUID, primary key), recommendation_id (UUID, foreign key to recommendations), engine_level (integer — 1, 2, 3, or 4), persona_prompt_version (text, e.g. 'v1.0'), model_name (text, e.g. 'claude-sonnet-4-20250514'), model_provider (text — anthropic or openai), generated_at (timestamptz, default now()).
Addition 3: Add three fields to the users table: deletion_requested_at (timestamptz, nullable), purged_at (timestamptz, nullable), purge_status (text, nullable — values: pending, scheduled, complete).
Also create a new table called consent_events. Fields: id (UUID, primary key), user_id (UUID, foreign key to users), consent_type (text — analytics, push_notifications), consent_given (boolean), occurred_at (timestamptz, default now()), ip_address_hash (text, nullable).
Enable RLS on all new tables. Apply permissive policies for now — the same approach as the existing tables.
