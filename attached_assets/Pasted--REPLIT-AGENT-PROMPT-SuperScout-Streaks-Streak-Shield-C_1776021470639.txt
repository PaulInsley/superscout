# REPLIT AGENT PROMPT — SuperScout Streaks + Streak Shield

## Context

SuperScout is a personality-driven AI fantasy football coach built in React Native with Supabase backend. Streaks are a Phase 2 retention feature inspired by Duolingo’s streak mechanic. The psychology is loss aversion — losing a long streak feels twice as painful as the pleasure of building one. After about 30 days, the streak becomes part of the user’s identity. This is one of the highest-impact, lowest-effort retention features in the product.

The streaks table already exists in the Supabase schema (Table 8). Fields: id, user_id, current_streak, longest_streak, streak_shield_available, season, last_active_gameweek.

## Task

Build the complete Streak system. Four parts: data logic, UI display, Streak Shield, and milestone celebrations.

-----

## PART 1 — Streak Data Logic

1. **What counts as “active”:** A user is active in a gameweek if they made at least one decision in SuperScout during that gameweek. A “decision” means any of: viewing captain recommendations, viewing transfer recommendations, generating a squad card, or confirming “no changes” (if that feature exists). Check the `decision_log` table for any entry matching this user and gameweek. Do NOT require the user to have actually made a transfer or changed captain on the FPL site — just engaging with SuperScout counts.
1. **When streaks update:** After each gameweek deadline passes, run a streak check for all users. For each user:
- Look up their `last_active_gameweek` from the streaks table
- Compare it to the gameweek that just ended
- If `last_active_gameweek` equals the gameweek that just ended → streak continues. Increment `current_streak` by 1. Update `last_active_gameweek`. If `current_streak` > `longest_streak`, update `longest_streak`.
- If `last_active_gameweek` is one gameweek behind (i.e. they missed this gameweek) → check Streak Shield (see Part 3). If shield available, use it. If not, reset `current_streak` to 0.
- If `last_active_gameweek` is more than one gameweek behind → streak is broken regardless. Reset `current_streak` to 0.
1. **New user initialisation:** When a user signs up, create a streaks row with current_streak = 0, longest_streak = 0, streak_shield_available = false, last_active_gameweek = null.
1. **Season reset:** Streaks are per-season (the `season` field handles this). At the start of a new season, create a fresh streaks row. Previous season data stays for historical reference.
1. **Create a server-side function** (Supabase Edge Function or server route) called `updateStreaks` that runs the streak check logic for all users. This will eventually be triggered automatically after each gameweek, but for now it can be triggered manually for testing.

- The home/dashboard screen (visible immediately on app open)
- The Settings screen (in the user profile section)
   
   The badge should show a flame/fire icon (use a suitable icon from the existing icon set or Feather icons) with the streak number next to it. Example: 🔥 7

- 0 streak: greyed out flame, no number

- 10–19 streaks: bright flame with subtle glow effect

## PART 3 — Streak Shield

1. **How it works:** The Streak Shield is a one-time protection per season that saves the user’s streak if they miss exactly one gameweek. Think of it like Duolingo’s Streak Freeze.
1. **Earning it:** The Streak Shield becomes available when the user reaches a 5-gameweek streak. Set `streak_shield_available` to true when `current_streak` reaches 5 for the first time in the season.
1. **Using it:** When the streak check (Part 1) finds that a user missed exactly one gameweek AND their `streak_shield_available` is true:
- Keep the streak alive (do NOT reset current_streak)
- Set `streak_shield_available` to false
- Log the shield usage (add a `streak_shield_used_gw` integer field to the streaks table to record which gameweek it was used in)
- Next time the user opens the app, show a notification: “Your Streak Shield saved your [N]-gameweek streak! You missed GW[X] but we’ve got your back. The shield is used — build another 5-week streak to earn a new one.”
1. **Re-earning:** After using the shield, it can be re-earned by building another 5-gameweek streak from the current point. When `current_streak` reaches 5 again after shield usage, set `streak_shield_available` back to true.
1. **UI indicator:** Show the Streak Shield status as a small shield icon next to the streak flame:
- Shield available: solid shield icon (gold/amber colour)
- Shield used: empty/outline shield icon (greyed out)
- Shield locked (streak < 5): no shield icon shown

## Architecture Note

The streak system must be sport-agnostic. The streaks table already has a `season` field. Add a `sport` field (text, default ‘fpl’) so that when F1 Fantasy is added, a user can have separate streaks per sport. The streak logic (consecutive rounds of activity, shield mechanic, milestones) works identically regardless of sport — only the round/gameweek terminology and season length change. Keep all FPL-specific text (like “gameweek” and “38 GWs”) in a sport-specific config, not hardcoded in the components.
