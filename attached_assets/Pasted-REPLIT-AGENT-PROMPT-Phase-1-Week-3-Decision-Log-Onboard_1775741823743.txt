REPLIT AGENT PROMPT — Phase 1 Week 3: Decision Log + Onboarding
We are starting Phase 1. The goal this week is to build two things: the silent Decision Log that records every recommendation and user choice into Supabase, and the onboarding flow that gets a new user set up in under 3 minutes.
Before you start, read replit.md to remind yourself of the project structure.
Do not run any destructive git operations without telling me first.

PART 1 — THE DECISION LOG
This is the most important thing in the entire product. Every downstream feature — What If analysis, Report Card, Manager Profile, Season Wrapped, Intelligence Engine learning — depends on this data existing from Gameweek 1. It must be silent and automatic. The user never sees it happening.
Create a file at artifacts/superscout/services/decisionLog.ts.
This file should export the following functions:
logRecommendation — called every time the AI generates a recommendation. It should write to three tables in sequence:
First, insert a row into recommendations with: user_id, gameweek, season (hardcode '2026-27' for now), decision_type (one of: 'captain', 'transfer', 'chip_usage', 'lineup'), options_shown (the full JSON of all options presented), persona_used, tier_at_time. Return the inserted recommendation's id.
Second, for each option shown, insert a row into recommendation_options with: recommendation_id (from above), option_rank (1, 2, 3 etc), player_id (if applicable), option_type, expected_points, confidence_score, confidence_label ('HIGH', 'MEDIUM', or 'SPECULATIVE'), upside_text, risk_text, is_superscout_pick (true for the SuperScout Pick, false for others).
Third, insert a row into inference_context with: recommendation_id, engine_level (1 for now — FPL API only), persona_prompt_version ('v1.0'), model_name ('claude-sonnet-4-6'), model_provider ('anthropic').
logUserDecision — called when the user taps to select an option. It should insert a row into user_decisions with: recommendation_id, user_id, recommendation_option_id (the specific option they chose), chosen_option (text description), hours_before_deadline (calculate from current time vs next FPL deadline if known, otherwise null).
Both functions should use the Supabase client from services/supabase.ts. Both functions should fail silently — wrap everything in try/catch and log errors to console but never crash the app if logging fails. The user experience must never be affected by a logging failure.

PART 2 — ONBOARDING FLOW
Build a 5-screen onboarding flow that runs the first time a user opens the app. Once completed, set onboarding_completed to true in the users table and never show it again. Store onboarding completion state in AsyncStorage as a fallback.
The 5 screens in order:
Screen 1 — Welcome
Full screen. SuperScout logo and name at the top. Tagline: "Your AI fantasy football coach. Three personalities. Smarter decisions." A single button: "Let's go". No login required yet.
Screen 2 — Connect Your FPL Team
Ask for the user's FPL Manager ID. Same input as the existing squad screen — reuse that component. Explain in one line where to find their ID: "Find it in the FPL app under Points > your name". Validate it's a number. Fetch their team name from the FPL API and display it as confirmation ("Found: [Team Name] ✓"). Button: "That's my team". Allow skipping with a small "Skip for now" link.
Screen 3 — Choose Your Persona
Three cards, one per persona. Each card shows the persona name, a one-line description, and a short example response to the same scenario ("Haaland at home this week — what do you think?"). The three example responses should be hardcoded and written now — do not call the AI API during onboarding.
Expert example: "Haaland carries the highest floor this week. City's xG against this defence averages 2.1 over the last four meetings. The ownership risk of not captaining him is as real as the risk of a blank."
Critic example: "City at home against that defence. If you don't captain Haaland here I genuinely cannot help you. This is the least interesting captain decision of the gameweek."
Fanboy example: "HAALAND. HOME. AGAINST THAT DEFENCE. The xG numbers are basically a cheat code right now and I need you to be excited about this because I absolutely am. This is the pick!!"
User taps one card to select it. Selected card highlights. Button: "This is my vibe".
Screen 4 — What SuperScout Does
Three bullet points with icons, kept very short:

"Gives you 3-5 options per decision, not just one answer"
"Tracks your decisions and shows what would have happened"
"Learns how you play and gets sharper over time"

Button: "Got it, let's play"
Screen 5 — You're In
Show their team name if they connected in Screen 2, otherwise show "Welcome to SuperScout". One line: "Your first recommendation is ready." Single button: "See my squad". This takes them to the main app.

PART 3 — WIRE IT TOGETHER

Check AsyncStorage on app launch for onboarding completion. If not complete, show the onboarding flow before the main tabs.
After onboarding completes, save the chosen persona to AsyncStorage and to the users table in Supabase if the user is logged in.
The FPL Manager ID entered in Screen 2 should pre-populate the squad screen so the user doesn't have to enter it again.
