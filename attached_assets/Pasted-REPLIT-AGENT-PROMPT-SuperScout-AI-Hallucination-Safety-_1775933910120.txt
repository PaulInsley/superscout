REPLIT AGENT PROMPT — SuperScout AI Hallucination Safety Check

CONTEXT: SuperScout generates transfer and captain recommendations using Claude AI. The validation layer already filters out recommendations that reference non-existent players or violate budget/position constraints. But we need an additional safety net specifically for AI hallucination — cases where the AI invents player names, fabricates statistics, or references fixtures that don’t exist.

TASK: Add a hallucination check layer that runs after AI generation but before results are shown to the user.

1. CREATE a new service file: services/validation/hallucination-check.ts
1. The hallucination check should verify:
   a. PLAYER EXISTENCE — Every player name mentioned in the AI response must match a real player in the FPL API bootstrap data. Use fuzzy matching (the AI might say “Mohamed Salah” but the API has “M. Salah” as web_name). Match against both web_name and the combination of first_name + second_name.
   b. TEAM EXISTENCE — Every team name mentioned must match a real Premier League team.
   c. FIXTURE ACCURACY — If the AI says “Arsenal vs Chelsea at home”, verify that fixture actually exists in the upcoming gameweek(s) using the fixtures endpoint data.
   d. STAT PLAUSIBILITY — If the AI claims a player “has scored in 5 of the last 6”, flag (don’t block) any stat claims for manual review. We can’t verify every stat, but we can log them.
1. For each check:
- If player doesn’t exist → REMOVE that recommendation option and log a warning
- If team doesn’t exist → REMOVE that option and log a warning
- If fixture is wrong → CORRECT the fixture info from API data (don’t remove the rec)
- If stats seem implausible → LOG for review but don’t block
1. Wire this check into both the captain and transfer routes, running AFTER the AI response is parsed but BEFORE it’s returned to the client. Also wire it into the pre-generation pipeline if that exists.
1. If the hallucination check removes options and fewer than 2 remain for Pro users, trigger a retry (one retry only) with a note appended to the prompt: “IMPORTANT: Only reference players who currently exist in the FPL game. Do not invent player names or statistics.”

The existing fpl-validator.ts handles rule validation (budget, positions, etc). This new file handles factual accuracy. They are separate concerns.
