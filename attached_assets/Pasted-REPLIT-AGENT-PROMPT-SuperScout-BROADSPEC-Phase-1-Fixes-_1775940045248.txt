## FIX 1: Admin panel has hardcoded fallback password (S-001 — High)

In artifacts/api-server/src/routes/admin.ts, line 28 currently reads:
const ADMIN_PASSWORD = process.env.ADMIN_PASSWORD || ‘superscout-admin-2026’;

This means if the environment variable is not set, anyone who guesses ‘superscout-admin-2026’ gets admin access. Change this so:

- If ADMIN_PASSWORD environment variable is not set, the admin routes should return 503 “Admin panel not configured” for ALL requests
- Remove the hardcoded fallback password entirely
- The admin login page can still render, but any login attempt should fail with “Admin panel not configured — set ADMIN_PASSWORD environment variable”

## FIX 4: No timeout on Claude API calls (R-001 — Medium)

The FPL API calls have a 15-second timeout, but the Claude AI calls in the captain, transfer, and pre-generation routes have no timeout at all. If Claude’s API is slow or unresponsive, requests hang indefinitely, which could block all server workers on deadline day.

In the following files, add a 30-second timeout to the Anthropic client:

- artifacts/api-server/src/routes/captain.ts (function getClient, line 44)
- artifacts/api-server/src/routes/transfer.ts (wherever the Anthropic client is created)
- artifacts/api-server/src/routes/preGenerate.ts (function getClient)
- artifacts/api-server/src/routes/squadCard.ts (wherever the Anthropic client is created)

The Anthropic SDK supports timeout in the constructor:
new Anthropic({ apiKey: …, baseURL: …, timeout: 30000 })

If the timeout is hit, catch the error and return a user-friendly message like “SuperScout is taking longer than usual — please try again in a moment.”
