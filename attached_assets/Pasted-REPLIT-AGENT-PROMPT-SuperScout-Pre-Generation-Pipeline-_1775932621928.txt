REPLIT AGENT PROMPT — SuperScout Pre-Generation Pipeline
CONTEXT: SuperScout generates captain picks and transfer advice via AI (Claude API). Currently these are generated on-demand when the user opens the screen. On deadline day with high traffic, this could mean slow responses or API rate limits. The pre-generation pipeline generates recommendations in advance so they load instantly.
TASK: Build a pre-generation pipeline with these requirements:
	1.	NEW SERVER ENDPOINT: POST /api/pre-generate/:gameweek
	•	Protected by the same admin secret used in processDecisions (PROCESS_DECISIONS_SECRET)
	•	For each user in the database who has an FPL manager ID connected:
a. Generate captain picks for all 3 vibes (expert, critic, fanboy)
b. Generate transfer advice for all 3 vibes
	•	Store results in a new Supabase table: pre_generated_recommendations
	•	Fields: id (uuid), user_id, gameweek, season, decision_type (captain/transfer), vibe (expert/critic/fanboy), response_json (jsonb), generated_at (timestamptz), expires_at (timestamptz), used (boolean default false)
	•	Set expires_at to the gameweek deadline time
	•	Rate limit: process users sequentially with a 2-second delay between each to avoid hammering the Claude API
	2.	UPDATE CAPTAIN AND TRANSFER SCREENS:
	•	Before making a live AI call, check pre_generated_recommendations for a matching (user_id, gameweek, decision_type, vibe) record where used = false and expires_at > now()
	•	If found: use the pre-generated response, mark it as used = true, show instantly
	•	If not found: fall back to the existing live AI generation (no change to current behaviour)
	3.	The pre-generation endpoint should be callable via a simple curl command or scheduled cron job. It does NOT need a UI — it will be triggered manually before each deadline or via an external scheduler later.
	4.	Add a Supabase migration file for the new table at scripts/supabase-migration-pre-gen.sql with RLS enabled (same pattern as other tables).
IMPORTANT: Do not modify the existing captain or transfer AI generation logic. The pre-gen pipeline should use the exact same prompt-building functions — just call them ahead of time and cache the results.
