Replit Agent Prompt — SuperScout: Manager Squad Screen
This is the next feature for the SuperScout app. We already have the player list screen pulling from the FPL API. Now I need a screen where a user enters their FPL manager ID and sees their own squad, transfer history, and mini-league standings.
What to build:

A "My Squad" screen with an input field. At the top, a simple text input where the user types their FPL manager ID number (just a number, e.g. 13042160) and taps a "Load Squad" button. Once loaded, remember the ID so they don't have to type it again next time they open the app — store it locally on the device.
Display the manager's current squad (15 players). Use the FPL API endpoint /entry/{manager_id}/ to get basic manager info (team name, overall rank, total points). Then use /entry/{manager_id}/event/{current_event}/picks/ to get their 15 picked players for the current gameweek. Cross-reference each player's ID against the /bootstrap-static/ data you already have to display:

Player name
Position (GKP, DEF, MID, FWD — the element_type field maps to these)
Current price (from now_cost, divided by 10)
Form rating
Whether they're the captain or vice-captain (the picks endpoint includes is_captain and is_vice_captain flags)
Whether they're on the bench (positions 12–15 in the picks array are bench players)

Show the starting 11 first, then a clear separator, then the 4 bench players underneath. Highlight the captain with a (C) marker and vice-captain with a (V) marker.
Display the manager's basic info at the top of the squad. Above the player list, show:

Team name (from /entry/{manager_id}/ — the field is name)
Manager name (fields player_first_name and player_second_name)
Overall points (field summary_overall_points)
Overall rank (field summary_overall_rank — format this with commas, e.g. 1,234,567)
Current gameweek points (from the picks endpoint, field entry_history.points)


Display transfer history. Use the endpoint /entry/{manager_id}/transfers/ to get a list of recent transfers. Show the last 10 transfers with:

Gameweek the transfer was made
Player transferred in (name and price at time of transfer)
Player transferred out (name and price at time of transfer)

You'll need to cross-reference player IDs against the bootstrap-static data for names. Put this in a "Transfers" tab or section below the squad.
Display mini-league standings. Use /entry/{manager_id}/ to get the manager's classic leagues (the field leagues.classic is an array). For each league, show:

League name
The manager's rank in that league
Total number of entries in that league

Don't try to load every league's full standings yet — just show the summary from the entry endpoint. Put this in a "Leagues" tab or section.
Work out the current gameweek automatically. From the /bootstrap-static/ data, look at the events array. The current gameweek is the one where is_current is true. If no gameweek is currently active, use the one where is_next is true and subtract 1 (the most recently completed one). Use this to know which gameweek's picks to load. If a gameweek is in progress but not all matches have been played, that's fine — just show whatever data is available.
Navigation. Add a bottom tab bar so the user can switch between the "Players" screen (the one we already built) and this new "My Squad" screen. Keep it simple — two tabs for now.
Error handling. If someone enters an invalid manager ID (one that doesn't exist), show a friendly error: "Couldn't find that manager ID — double-check the number and try again." If the API is down, show the same style of error message we already have.
Architecture reminder. Keep all FPL API calls in the existing services/fpl/ folder (or wherever the FPL connector module lives from the first build). The new endpoints are still FPL-specific, so they belong in the same module. The squad display screen itself should be in the main app screens — it's the sport-specific data connector that stays isolated, not the UI.
